# 写在开头
我记得前今年主流的移动端配置都是 rem，包括最近我在面试别人的时候，大家给的答案也都是 rem。但时过境迁，我觉得应该会有比 rem 更好的做法。

不是说 rem 不好，实在是每次都要去转换单位这个事情比较繁琐，及时有插件帮你去转，例如 px2rem.但通篇的 px2rem() 也不好看。而且最主要的，无法兼容 tailwindcss.

所以，我一直在探索更加友好的写法。


# 探索
我查看了 [vant](https://vant-ui.github.io/vant/#/zh-CN/advanced-usage#liu-lan-qi-gua-pei) 的文档，他们采用的是[postcss-px-to-viewport](https://github.com/evrone/postcss-px-to-viewport) 这种，我试了下确实可以带来兼容（大家可以参考 vant 提供的写法），而且完美适配 tailwindcss。单独的移动端开发很推荐大家去使用这个方法。

但我有一个双端开发的项目，权衡再三后觉得应该把 web 端和 H5 端放在一个项目里（因为这个项目大部分的 js 代码是可以共用的，放在一起可以更高效的完成开发）。如何，在使用 postcss-px-to-viewport 就显得有点力不从心。虽然这个插件提供了配置去忽略适配一些组件甚至是 css 选择器。但实际开发中还是不够灵活。

# 再次探索
其实 viewport 做适配还是很简单的。这段是适用于 h5 端的

```js
;(function autoWidth() {
  let baseWidth = 375 // 基准宽度

  // 如果屏幕宽度不等于基准宽度，则进行调整
  if (screen.width !== baseWidth) {
    let viewportMeta = document.querySelector("meta[name='viewport']")
    let screenWidth = screen.width
    let scale = 1

    // 针对 1080p 屏幕设备的处理，将宽度调整为 360
    if (screenWidth === 1080) {
      screenWidth = 360
    }

    // 针对特定版本的 UC 浏览器进行处理，将宽度调整为 360
    if (/Android\s4\.3.*UCBrowser\/11\.1\.0\.870.*/.test(window.navigator.userAgent)) {
      screenWidth = 360
    }

    // 计算缩放比例
    scale = screenWidth / baseWidth

    // 更新 viewport meta 标签内容
    viewportMeta.content =
      'width=' + baseWidth + ',initial-scale=' + scale + ',maximum-scale=' + scale + ',minimum-scale=' + scale
  }
})()
```

如果想要适配双端，只需在屏幕大于 1200px 时不去适配即可。

```js
(function autoWidth() {
  let baseWidth = 375; // 基准宽度

  // 仅当屏幕宽度小于1200时执行
  if (screen.width >= 1200) {
    return;
  }

  // 如果屏幕宽度与基准宽度不同，则进行调整
  if (screen.width !== baseWidth) {
    let viewportMeta = document.querySelector("meta[name='viewport']");
    let screenWidth = screen.width;
    let scale = 1;

    // 针对1080p设备的特定处理
    if (screenWidth === 1080) {
      screenWidth = 360;
    }

    // 针对特定版本的UC浏览器进行处理
    if (
      /Android\s4\.3.*UCBrowser\/11\.1\.0\.870.*/.test(
        window.navigator.userAgent
      )
    ) {
      screenWidth = 360;
    }

    // 计算缩放比例
    scale = screenWidth / baseWidth;

    // 更新viewport meta标签内容
    viewportMeta.content =
      "width=" +
      baseWidth +
      ",initial-scale=" +
      scale +
      ",maximum-scale=" +
      scale +
      ",minimum-scale=" +
      scale;
  }
})();

```

# viewport 下的 bug

这种方法用了这么久，我确实也发现了一个适配的 bug。如果两张图片紧挨着排列，会在图片中间有一道透明的像素差。你可以通过背景填充，也可以通过 mt-[-1px] 、 ml-[-1px]这种方式去解决。
