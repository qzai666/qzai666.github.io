### 大家好

这是我的第一篇博文，也是这个博客的一次小改造记录。

最开始这个站点是用 [Hexo](https://hexo.io/zh-cn/) 搭起来的，主题用的是 [Icarus](https://github.com/ppoffice/hexo-theme-icarus)。当时主要目标很简单：先把博客跑起来，能写文章、能归档、能部署到 GitHub Pages 就行。

最近我又用 Codex 把它重新整理了一遍，主要做了几件事：

- 把站点语言从 `en` 改成 `zh-CN`，让主题里的默认文案更贴近中文博客。
- 把主题从 `icarus` 换成了 `fluid`，整体结构更适合做一个偏视觉化的个人技术博客。
- 新增 `_config.fluid.yml`，单独维护 Fluid 主题配置，避免把主题配置全部塞进主配置文件。
- 新增 `source/image/neon-cyberpunk.svg`，作为首页和文章页的霓虹赛博风头图。
- 新增 `source/css/neon-cyberpunk.css`，统一处理黑紫底色、青粉霓虹、高亮标题、扫描线、玻璃面板和代码块样式。
- 更新 `package.json`，加入 `hexo-theme-fluid` 依赖，让部署环境能稳定安装主题。

这次改造中，我先让 Codex 找了几种主题方向，包括科技感、暗黑极简、玻璃拟态和朋克风。最后我选了“霓虹赛博风”，所以整体视觉就往黑色背景、青色霓虹、粉紫光效和未来城市感上靠。

过程中也踩了一点小坑。比如一开始试过直接改 Icarus，后来发现这不是我想要的方向；又试了 Butterfly，但它更像卡片式内容博客，科技感不够强。最后换到 Fluid，首屏 Banner、暗色模式和文章列表结构都更适合继续定制。

留言功能也顺手重新整理了一下。最开始博客里没有评论区，后来先接入了 Fluid 支持的畅言评论。畅言能正常显示，但实际使用时问题比较明显：暗色主题下输入框文字发黑、发送按钮文字看不清，而且点击发布后还要求手机号登录，这和博客本身的技术分享氛围不太搭。

为了解决这个问题，Codex 先针对畅言的 DOM 写了一版样式，把输入框、提示文字和按钮都调整成霓虹暗色风格。但样式只是解决了“看得见”的问题，手机号登录这个门槛还是存在。

后来又试了 `utterances`，它基于 GitHub Issues，不需要手机号，视觉上也比畅言更适合技术博客。不过本地验证时发现它要求给仓库安装 utterances GitHub App，否则留言时会报错：

```text
Error: utterances is not installed on qzai666/qzai666.github.io.
```

我又尝试过直接把当前仓库的 GitHub Issues 当成留言入口，文章底部显示“去 GitHub 留言”和“查看已有留言”两个按钮。这个方案代码很简单，但后来发现一个关键问题：当前仓库是私有仓库，普通访客没有权限访问 Issues，所以它并不适合做公开留言区。

因此最后把评论系统切到 `Cusdis`。它有免费方案，访客留言不需要手机号、GitHub 或第三方账号登录，也不依赖当前私有仓库的 Issues 权限。Fluid 本身已经内置 Cusdis 支持，所以配置上只需要把评论类型改成 `cusdis`，再补上 Cusdis 后台生成的 `app_id`。在 `app_id` 填好之前，文章底部先显示一个配置提示，避免评论区直接空白。

改完之后，Codex 还帮我做了本地验证：

```bash
npm run build
npx hexo server -p 4020
```

然后用 Playwright CLI 分别截了桌面端和移动端页面，检查有没有白屏、文字重叠、移动端溢出这些明显问题。现在这个版本至少已经能正常构建，也能在本地预览里看到完整的霓虹赛博风效果。

以前搭博客更多是“能用就行”，现在有了 Codex 之后，改主题、查配置、写样式、跑构建、做截图验证都可以串起来完成。这个过程比手动一点点翻主题文档快很多。

后面这个博客会继续用来记录前端、AI 编程、浏览器自动化和一些工程化实践。学习是终身的修养，一起加油吧。
